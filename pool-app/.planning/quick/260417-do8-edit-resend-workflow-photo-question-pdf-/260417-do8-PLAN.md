## Background (context captured from exploration)

### Submission/status model
- `JobStatus`: `DRAFT | SUBMITTED | ARCHIVED`.
- `src/app/jobs/[id]/page.tsx:46` treats SUBMITTED and ARCHIVED as read-only — disables form, hides SubmitSection, hides PhotoUpload.
- `src/lib/actions/submit.ts:55-57` fast-path rejects `status === "SUBMITTED"`.
- `src/lib/actions/submit.ts:125-136` atomic `updateMany` with `status: { not: "SUBMITTED" }` guard. Sets `status=SUBMITTED, submittedBy, submittedAt, workerSignature`.
- If we set status back to DRAFT, the existing submit flow regenerates PDF + resends email on resubmit — **no changes needed in submit.ts**.

### Photo model (the mismatch)
- Form-field photos (`src/components/job-form.tsx:277-304`): upload → `savePhotoMetadata()` appends to `job.photos[]` AND `rhf.onChange(url)` writes URL to `formData[fieldId]`. So URL appears in **both** places, field-bound.
- Gallery-only photos (`src/components/photo-upload.tsx` via `src/app/jobs/[id]/page.tsx:106`): only land in `job.photos[]`, no `formData[fieldId]` binding.
- PDF generator `src/lib/actions/generate-pdf.ts:168-179`: builds `photosQueue` = `job.photos` URLs NOT in any field's `formData`. i.e. the gallery-only extras.
- PDF generator `:263`: non-Q108 photo fields `const photoUrl = directUrl ?? photosQueue.shift() ?? null;` — **pops a gallery photo into an empty field slot**. That's the mismatch. A gallery photo uploaded to supplement the job appears as if it were answering question N.
- Q108 `:207-256`: drains remaining queue — correct behavior (these are truly "additional").

### Logo line artifact
- `src/lib/actions/generate-pdf.ts:111-115`:
  ```ts

  doc.setLineWidth(0.5);
  doc.line(MARGIN, y, PAGE_WIDTH - MARGIN, y);
  ```

### Admin / PIN
- Admin page `src/app/admin/page.tsx` unguarded route; actions PIN-gated.
- PIN verified via `checkPin()` → `src/lib/actions/settings.ts:24-28`.

### 1a. Add `reopenJob` server action

Append to `src/lib/actions/settings.ts` (after `archiveJob`, before `deleteJob`):

```ts
export async function reopenJob(
  pin: string,
  jobId: string
): Promise<{ success: boolean; error?: string }> {
  if (!(await checkPin(pin))) {
    return { success: false, error: "Incorrect PIN" };
  }

  const job = await db.job.findUnique({ where: { id: jobId } });
  if (!job) {
    return { success: false, error: "Job not found" };
  }
  if (job.status !== "SUBMITTED") {
    return { success: false, error: "Only submitted jobs can be reopened" };
  }

  // Move back to DRAFT and clear submission fingerprint so the status badge,
  // "Submitted by" header, and signature block all reflect a fresh draft.
  // Photos and formData are preserved so the office only has to edit what changed.
  await db.job.update({
    where: { id: jobId },
    data: {
      status: "DRAFT",
      submittedBy: null,
      submittedAt: null,
      workerSignature: null,
    },
  });

  revalidatePath("/");
  revalidatePath("/admin");
  revalidatePath(`/jobs/${jobId}`);
  return { success: true };
}
```

  ```ts
  async function handleReopen(jobId: string) {
  ```
