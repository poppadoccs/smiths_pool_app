# Quick Task 260416-ki0: Embed Job Photos in PDF as Photo Appendix

## Context

`generate-pdf.ts` currently renders photo fields as `"(photo attached)"` placeholder text.
Job photos are stored in `job.photos` as a JSON array of `PhotoMetadata { url, filename, size, uploadedAt }`.
All photos uploaded through the app are JPEG (browser-image-compression forces `fileType: "image/jpeg"`).
The PDF is generated server-side where `fetch` is available to download Vercel Blob URLs.

## Task 1: Add photo appendix to generate-pdf.ts

**File:** `src/lib/actions/generate-pdf.ts`

**Action:**
1. Import `PhotoMetadata` from `@/lib/photos`
2. After the worker signature block (end of current PDF), check `job.photos`:
   - If empty / null, do nothing (no appendix)
   - If photos exist, add `Photo Appendix` section:
     a. Add new page
     b. Print section header "Photo Appendix" in bold 12pt
     c. Print sub-line: "Job photos attached at time of submission"
     d. For each photo:
        - `fetch(photo.url)` → `arrayBuffer()` → `Buffer.from(...).toString("base64")`
        - Try `doc.getImageProperties(base64)` to get real width/height for aspect ratio
        - Compute rendered height: `(imgProps.height / imgProps.width) * CONTENT_WIDTH`
        - Cap rendered height at 180mm to prevent single portrait photo filling entire page
        - If `y + renderedHeight + 10 > 280`, insert new page and reset y
        - Print label: `Photo N of M: ${photo.filename}` in bold 9pt
        - `doc.addImage(base64, "JPEG", MARGIN, y, CONTENT_WIDTH, renderedHeight, undefined, "FAST")`
        - Advance `y += renderedHeight + 8`
        - On error (fetch or getImageProperties): print italic placeholder `"[Photo could not be loaded: filename]"` and continue

**done:** `job.photos` array is fully iterated, each photo either rendered or gracefully skipped

**verify:**
- Submit a job with ≥1 photo and call `generateJobPdf` — returned base64 PDF should be noticeably larger than a photo-less job
- Open PDF viewer: last pages should contain real photos, not placeholder text
- Jobs with no photos: PDF unchanged (no blank appendix page)

## Notes

- All job photos are JPEG — use `"JPEG"` format in `addImage` unconditionally (no extension sniffing needed)
- `COMPRESSION: "FAST"` in `addImage` reduces file size at acceptable quality for a field report
- `getImageProperties` may throw if base64 is malformed; wrap in try/catch around the whole photo block
- Do NOT change the `"(photo attached)"` rendering in the form-fields section — that's accurate context and keeps the form layout clean; photos appear in the appendix
